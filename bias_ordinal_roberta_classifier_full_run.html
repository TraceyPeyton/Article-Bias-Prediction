Ordinal Transformer Classifier for Article Bias Prediction¶

This notebook revises the original LSTM approach in response to the review comments, but it is configured to be crash-safe by default:

  • Treat political bias as an ordered spectrum: left < center < right.
  • Use a RoBERTa-family transformer for contextual embeddings instead of an LSTM trained from scratch.
  • Use cumulative ordinal targets, where left = (1, 0, 0), center = (1, 1, 0), and right = (1, 1, 1), with binary cross-entropy over the ordinal outputs.
  • Report both standard classification metrics and distance-aware metrics so left/right mistakes are penalized more than left/center or center/right mistakes.

The default settings run a small smoke test to avoid exhausting memory. Set RUN_FULL_TRAINING = True in the configuration cell when you are ready for the full experiment.

2. Ordinal Cumulative Encoding¶

The labels are not independent categories; they sit on a political spectrum. The cumulative target below follows the review suggestion:

  • left = (1, 0, 0)
  • center = (1, 1, 0)
  • right = (1, 1, 1)

At inference time, the predicted class is recovered by thresholding each sigmoid output at 0.5 and counting the active cumulative bits.

In [3]:
def to_ordinal_targets(label_ids, num_classes=3):
    label_ids = np.asarray(label_ids, dtype=np.int64)
    thresholds = np.arange(num_classes, dtype=np.int64)
    return (label_ids[:, None] >= thresholds[None, :]).astype(np.float32)


def ordinal_probs_to_labels(probabilities, threshold=0.5):
    active_bits = (probabilities >= threshold).astype(np.int64).sum(axis=1)
    # left has one active bit, center two, right three.
    return np.clip(active_bits - 1, 0, len(LABEL_ORDER) - 1)

def stratified_sample(frame, n, seed=SEED):
    if n is None or len(frame) <= n:
        return frame.reset_index(drop=True)
    per_class = max(1, n // len(LABEL_ORDER))
    sampled = (
        frame.groupby('label', group_keys=False)
             .apply(lambda group: group.sample(min(len(group), per_class), random_state=seed))
             .sample(frac=1, random_state=seed)
             .reset_index(drop=True)
    )
    return sampled

class ArticleBiasOrdinalDataset(Dataset):
    def __init__(self, frame, tokenizer, max_length=128):
        self.frame = frame.reset_index(drop=True)
        self.tokenizer = tokenizer
        self.max_length = max_length
        self.ordinal_targets = to_ordinal_targets(self.frame['label'].values)

    def __len__(self):
        return len(self.frame)

    def __getitem__(self, idx):
        row = self.frame.iloc[idx]
        encoded = self.tokenizer(
            row['text'],
            truncation=True,
            max_length=self.max_length,
            padding='max_length',
            return_tensors='pt',
        )
        item = {key: value.squeeze(0) for key, value in encoded.items()}
        item['labels'] = torch.tensor(self.ordinal_targets[idx], dtype=torch.float32)
        item['class_label'] = torch.tensor(int(row['label']), dtype=torch.long)
        return item

4. RoBERTa With Ordinal Head¶

The model outputs three ordinal logits instead of a three-way softmax. Binary cross-entropy trains each cumulative decision, so a far-off error usually requires more wrong ordinal decisions than an adjacent-class error.

In [5]:
class OrdinalRobertaClassifier(nn.Module):
    def __init__(self, model_name, num_ordinal_outputs=3, dropout=0.1, freeze_encoder=False):
        super().__init__()
        self.config = AutoConfig.from_pretrained(model_name)
        self.encoder = AutoModel.from_pretrained(model_name, config=self.config)
        hidden_size = self.config.hidden_size
        self.dropout = nn.Dropout(dropout)
        self.classifier = nn.Linear(hidden_size, num_ordinal_outputs)
        self.loss_fn = nn.BCEWithLogitsLoss()

        if freeze_encoder:
            for parameter in self.encoder.parameters():
                parameter.requires_grad = False

    def forward(self, input_ids=None, attention_mask=None, labels=None, **kwargs):
        outputs = self.encoder(input_ids=input_ids, attention_mask=attention_mask, **kwargs)
        cls_embedding = outputs.last_hidden_state[:, 0]
        logits = self.classifier(self.dropout(cls_embedding))

        loss = None
        if labels is not None:
            loss = self.loss_fn(logits, labels.float())

        return {'loss': loss, 'logits': logits}

5. Distance-Aware Metrics¶

Standard accuracy is still useful, but it treats every mistake equally. Mean absolute error on class IDs gives adjacent mistakes a penalty of 1 and left-right mistakes a penalty of 2. Squared distance makes far-off errors even more visible.

In [6]:
def compute_metrics(eval_pred):
    logits, labels = eval_pred
    probabilities = 1 / (1 + np.exp(-logits))
    y_pred = ordinal_probs_to_labels(probabilities)
    y_true = ordinal_probs_to_labels(labels)

    return {
        'accuracy': accuracy_score(y_true, y_pred),
        'ordinal_mae': mean_absolute_error(y_true, y_pred),
        'ordinal_mse': mean_squared_error(y_true, y_pred),
        'severe_error_rate': float(np.mean(np.abs(y_true - y_pred) == 2)),
    }

10. Performance Visualizations and Conclusions¶

The full ordinal transformer run improves substantially over the smoke tests and produces balanced F1 scores around 0.71-0.72 across all three ideological labels. The ordinal metrics add useful context beyond accuracy: the model's mean absolute ideological distance is 0.39 on a 0-2 scale, and 10.9% of test predictions are severe left-right/right-left errors.

The confusion matrix shows that the model is strongest at identifying center articles and generally separates the extremes, but right articles are sometimes pulled toward center. That pattern is preferable to many direct right-left confusions, yet the severe-error rate indicates that more work is still needed before using the model in a high-stakes setting. Next improvements would be threshold tuning for the ordinal bits, class-specific calibration, and comparing this cumulative head against a standard RoBERTa softmax baseline.

8. Prediction Helper¶

The helper returns cumulative probabilities, the decoded class, confidence, and the ordinal score. The ordinal score can also be interpreted as a continuous left-to-right position.

In [9]:
def predict_bias(text):
    model.eval()
    encoded = tokenizer(
        text,
        truncation=True,
        max_length=MAX_LENGTH,
        padding='max_length',
        return_tensors='pt',
    )
    device = next(model.parameters()).device
    encoded = {key: value.to(device) for key, value in encoded.items()}

    with torch.no_grad():
        logits = model(**encoded)['logits']
        probs = torch.sigmoid(logits).cpu().numpy()[0]

    predicted_id = int(ordinal_probs_to_labels(probs[None, :])[0])
    ordinal_score = float(np.clip(probs.sum() - 1, 0, 2))

    return {
        'predicted_bias': ID_TO_LABEL[predicted_id],
        'predicted_id': predicted_id,
        'ordinal_score_left_to_right': ordinal_score,
        'cumulative_probabilities': {f'bit_{i}': float(value) for i, value in enumerate(probs)},
    }

11. Baseline vs Updated Model Comparison¶

This section compares the original presentation's BiLSTM baseline against the updated ordinal DistilRoBERTa model. The transformer model improves test accuracy from 61.4% to 71.9%, with the largest F1 gain on center articles. It also adds distance-aware evaluation: ordinal MAE is 0.390 on the 0=left to 2=right scale, and severe opposite-pole errors occur in 10.9% of test cases.

On the newly scraped articles, the updated model changes the main source-level story. The old LSTM report found FOX was often predicted left; the new ordinal RoBERTa report predicts FOX mostly right. This suggests the transformer upgrade reduced the most visible out-of-distribution failure from the baseline, though these are still model predictions rather than ground-truth article labels.
